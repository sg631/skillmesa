## Blocked

### Deploying Cloud Functions

`firebase deploy --only functions` fails with an IAM policy error. The Firebase project requires 3 IAM bindings that only a **Project Owner** can set.

**Required commands (must be run by the project owner):**

```bash
gcloud projects add-iam-policy-binding skill-mesa \
  --member=serviceAccount:service-974991188890@gcp-sa-pubsub.iam.gserviceaccount.com \
  --role=roles/iam.serviceAccountTokenCreator

gcloud projects add-iam-policy-binding skill-mesa \
  --member=serviceAccount:974991188890-compute@developer.gserviceaccount.com \
  --role=roles/run.invoker

gcloud projects add-iam-policy-binding skill-mesa \
  --member=serviceAccount:974991188890-compute@developer.gserviceaccount.com \
  --role=roles/eventarc.eventReceiver
```

The project owner needs the [gcloud CLI](https://cloud.google.com/sdk/docs/install) installed and authenticated (`gcloud auth login`).

### Setting Algolia Admin Key as Firebase Secret

```bash
firebase functions:secrets:set ALGOLIA_ADMIN_KEY
```

Paste the Algolia Admin API key when prompted (found in Algolia Dashboard > Settings > API Keys).

### Running the Backfill Script

After functions are deployed, seed the Algolia index with existing listings:

```bash
export GOOGLE_APPLICATION_CREDENTIALS=/path/to/skill-mesa-serviceaccount.json
cd functions && ALGOLIA_ADMIN_KEY=<your-admin-key> node scripts/backfill.js
```

The service account JSON is generated by the project owner via:
Firebase Console > Project Settings > Service accounts > "Generate new private key"
